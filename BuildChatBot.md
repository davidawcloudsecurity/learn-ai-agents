# Something
```
This lab exercise will step you through the end-to-end development flow via chatting with your selected
coding assistant:
1. Feature description (Planning) - Describe the feature or task to implement. The feature description
should be detailed and clear enough to ensure that the assistant can build a clear set of artifacts for
the feature. If there are specific interface / API details or technology choices that the system should
adhere to, this is where they should be specified.
2. Requirements gathering – Requirements gathering articulates the system requirements in a way
that can be tracked through to completion. You can add or adjust the requirements interactively with
the assistant until the requirements are clear.
3. Design document - Once the requirements are finalized, A detailed design document should be
created that includes important design choices and architectural description of the envisioned feature
implementation. You can add or adjust the design interactively with the assistant until the design is
clear.
4. Tasks list (coding) - Once the requirements and design are finalized, create a set of tasks that outline
the implementation plan for the feature. As with the previous steps you can add or adjust the tasks
list interactively with the assistant until the task list is complete.
5. Testing – Once your feature implementation is complete your coding assistant should be able to help
you test using the provided feature validation script
6. Deployment – This is handled via the burner application pipeline
7. Maintenance – This step isn’t used for this lab exercise
```
